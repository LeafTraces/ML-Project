## 1. 项目一句话概括

本项目完成的是视网膜眼底图像中的渗出液病灶分割。输入是一张彩色眼底图像，输出是每个像素属于渗出液的概率或二值掩膜。项目比较了传统机器学习方法和深度学习方法：

- 传统机器学习：图像预处理 + 手工特征 + SVM / Logistic Regression 像素分类。
- 深度学习：使用 ImageNet 预训练 ResNet34 编码器的 U-Net 语义分割模型。

最终实验表明，U-Net 在 Dice、IoU、PR-AUC、ROC-AUC 和召回率上明显优于传统机器学习模型，但训练时间也更长。SVM 和 Logistic Regression 的训练速度很快，适合作为轻量基线，但在极端类别不平衡的病灶分割任务中容易出现漏检或误检。

## 2. 项目背景与任务要求

题目背景来自 `topic/眼底图像的渗出液分割/眼底图像的渗出液分割.docx`。题目指出：

- 数据来自公开 e-ophtha 数据集。
- 数据集中共有 47 张已标注渗出液图像。
- 眼底渗出液是糖尿病视网膜病变等眼部疾病的重要病灶之一。
- 传统监督学习方法通常包括预处理、候选病灶检测、特征提取、分类。
- 深度学习方法通常把特征选择和分类交给神经网络自动完成，但更依赖数据规模和计算资源。
- 作业要求选择两种机器学习方法进行渗出液分割，并给出 ROC 或 PR 曲线、IoU 或 Dice 等评价指标，还需要比较学习时间、准确率、召回率和混淆矩阵。

本项目实际完成了三组模型结果：

- SVM，作为传统机器学习分类器。
- Logistic Regression，作为另一个传统机器学习分类器。
- U-Net，作为深度学习分割模型。

答辩时可以把 SVM 和 Logistic Regression 归为“传统机器学习方法”，把 U-Net 归为“深度学习方法”。这样既满足“两种机器学习方法”的要求，又多提供了一个传统基线用于横向比较。

## 6. 预处理算法细节

预处理在 `src/exudate_data.py` 中完成。它的目标是减轻眼底图像中光照不均、黑色背景、对比度低等问题，让后续模型更专注于视网膜内的亮黄色渗出液病灶。

### 6.1 图像和掩膜读取

每个样本通过 `load_sample()` 读取：

- 彩色图像用 OpenCV 读取，再从 BGR 转成 RGB。
- 掩膜用灰度图读取。
- 图像 resize 到工作尺寸 `672 x 448`。
- 掩膜用最近邻插值 resize，避免把类别标签插成灰色过渡。
- 掩膜最终二值化为 `mask > 127`。

这里最近邻插值很重要。分割掩膜是类别标签，不是自然图像，如果用双线性插值，会产生中间灰度值，破坏前景/背景边界。

### 6.2 FOV 有效视野提取

眼底图像通常有黑色背景，模型不应该在黑色背景区域计算指标或训练分类器。因此代码用 `compute_fov_mask()` 提取有效视野。

步骤如下：

1. 把 RGB 图转成灰度图。
2. 在非零像素中计算平均灰度。
3. 阈值设为 `max(8, 0.5 * mean(gray > 0))`。
4. 灰度高于阈值的区域作为初始视野候选。
5. 用 `15 x 15` 椭圆结构元素做形态学开运算和闭运算。
6. 连通域分析，只保留面积最大的连通区域。
7. 用 flood fill 填补内部孔洞。
8. 返回二值 FOV mask。

形态学开运算可以去掉小噪声，闭运算可以填平边缘的小缺口。保留最大连通域可以防止噪点被当成视网膜区域。

### 6.3 光照归一化

眼底图像常有中心亮、边缘暗，或者局部光照不均的问题。代码中的 `remove_illumination()` 对每个颜色通道分别处理：

1. 使用中值滤波估计背景光照。
2. 中值滤波核大小为 `min(height, width) // 12` 后调整为奇数。
3. 对每个像素做：

```text
corrected = original - background + center
```

其中 `center` 是 FOV 内该通道的平均亮度。

这样做的直觉是：如果某处整体偏亮，背景估计值也高，减去背景后会被压下来；如果某处整体偏暗，背景估计值低，减去背景后会被抬起来。最后加回全局中心亮度，使图像整体亮度保持在合理范围。

### 6.4 CLAHE 对比度增强

代码中的 `apply_clahe()` 会：

1. 把 RGB 转成 LAB。
2. 只对 L 通道，即亮度通道，执行 CLAHE。
3. 再把 LAB 转回 RGB。

CLAHE 是 Contrast Limited Adaptive Histogram Equalization。它不是对整张图做统一直方图均衡，而是在局部窗口中增强对比度，并通过 clip limit 限制噪声过度放大。本项目中参数为：

```python
cv2.createCLAHE(2.0, (8, 8))
```

这一步有利于增强渗出液和周围组织之间的亮度差异。

## 7. 传统机器学习算法细节

传统机器学习部分在 `src/traditional_ml.py` 中实现。它把分割问题转化为像素级二分类问题：

```text
每个 FOV 内像素 -> 提取 14 维特征 -> 分类为渗出液或非渗出液
```

### 7.1 候选检测与 Top-hat 思路

代码中定义了 `candidate_mask()`，用于形态学候选检测和候选召回分析。

步骤如下：

1. 把 RGB 转为 LAB。
2. 取 L 通道作为亮度图。
3. 使用 `15 x 15` 椭圆结构元素做 Top-hat 变换。
4. 在 FOV 内计算 Top-hat 响应的均值和标准差。
5. 用 `mean + std` 作为阈值，提取亮小区域。
6. 使用 `3 x 3` 椭圆结构元素开运算去掉噪点。

Top-hat 变换的定义可以理解为：

```text
Top-hat(image) = image - opening(image)
```

开运算会去掉比结构元素小的亮结构，因此原图减去开运算结果后，会突出小而亮的病灶候选。眼底渗出液通常呈亮黄色或亮白色斑块，因此 Top-hat 对它有一定敏感性。

Notebook 完整运行日志中，候选检测召回上限为：

```text
候选检测召回上限: 0.557
```

这表示仅靠这个形态学候选检测，大约能覆盖 55.7% 的真实渗出液前景像素。它不是最终模型的性能，而是一个中间诊断量：说明单纯依赖亮度候选还不够，需要后续分类器或深度网络进一步学习。

需要注意：当前训练函数没有把 `candidate_mask` 作为硬过滤条件来丢弃所有非候选像素。最终 SVM/LR 是在 FOV 内抽样像素并提取特征训练，Top-hat 更多作为特征和可解释的候选分析存在。

### 7.2 14 维手工特征

`feature_maps()` 为每个像素构造 14 维特征：

| 序号 | 特征名 | 含义 |
|---:|---|---|
| 1 | R | RGB 的红通道，归一化到 0-1 |
| 2 | G | RGB 的绿通道，归一化到 0-1 |
| 3 | B | RGB 的蓝通道，归一化到 0-1 |
| 4 | L | LAB 亮度通道 |
| 5 | a | LAB 的 a 通道，表示绿-红方向 |
| 6 | b | LAB 的 b 通道，表示蓝-黄方向 |
| 7 | S | HSV 饱和度 |
| 8 | V | HSV 亮度 |
| 9 | contrast9 | L 通道减去 9 邻域均值 |
| 10 | contrast25 | L 通道减去 25 邻域均值 |
| 11 | localStd | 9 邻域局部标准差 |
| 12 | tophat9 | 9 尺度 Top-hat 响应 |
| 13 | tophat21 | 21 尺度 Top-hat 响应 |
| 14 | gradMag | Sobel 梯度幅值 |

这些特征覆盖了三类信息：

- 颜色信息：RGB、LAB、HSV。
- 局部亮度对比：contrast9、contrast25、localStd。
- 形态和边缘信息：Top-hat、Sobel 梯度。

渗出液通常颜色偏亮、偏黄，并且与周围组织有一定局部对比。因此 LAB 的 L、b 通道，RGB 的 B/G 通道，以及 V、Top-hat 等特征在图中系数较高。

从 `fig2_feat_coefficients.png` 可见：

- Logistic Regression 对 L、B、b、G、a 等颜色和亮度特征权重较大。
- SVM 也明显依赖 B、L、b、V 等特征。
- `contrast25` 权重较低，说明大尺度均值差异在当前线性模型里贡献较小。

### 7.3 类别不平衡处理

原始图像中前景只有约 0.2%。如果直接把所有 FOV 像素拿来训练，负样本会淹没正样本。代码采用两种策略：

1. 负样本下采样：

```python
neg_per_pos = 15
```

对每张训练图像，保留全部正样本，同时最多采样 `15 x 正样本数` 的负样本。

2. 分类器内部使用类别权重：

```python
class_weight = "balanced"
```

这样少数类前景像素在损失中权重更高。

Notebook 日志显示传统 ML 训练矩阵为：

```text
训练样本 (254448, 14)
正例率 0.062
```

也就是说，下采样后训练集正例率约为 6.2%，比原始 0.2% 高很多，更利于模型学习前景特征。

### 7.4 SVM 原理与实现

项目中使用的是线性支持向量机：

```python
LinearSVC(
    C=1.0,
    class_weight="balanced",
    dual=False,
    max_iter=5000,
    random_state=seed,
)
```

SVM 的目标是在特征空间中找到一个分类超平面：

```text
w^T x + b = 0
```

它希望正负样本被分到超平面两侧，同时让离超平面最近的样本，即支持向量，到超平面的间隔尽可能大。线性 SVM 的优化目标可以概括为：

```text
minimize  1/2 ||w||^2 + C * sum(max(0, 1 - y_i (w^T x_i + b)))
```

其中：

- `1/2 ||w||^2` 让模型更简单，间隔更大。
- `max(0, 1 - y_i f(x_i))` 是 hinge loss。
- `C` 控制分类错误和间隔大小之间的权衡。
- `class_weight="balanced"` 会提高少数类前景像素的损失权重。

因为 `LinearSVC` 默认输出的是 decision score，不是概率，所以项目额外使用：

```python
CalibratedClassifierCV(method="sigmoid", cv=3)
```

Sigmoid calibration 会学习一个单调映射，把 SVM 的 decision score 转成近似概率。这样后续才能统一画 ROC/PR 曲线，并在验证集上选择阈值。

### 7.5 Logistic Regression 原理与实现

项目中使用的逻辑回归配置为：

```python
LogisticRegression(
    class_weight="balanced",
    max_iter=1000,
    random_state=seed,
    solver="lbfgs",
)
```

逻辑回归也是线性分类器。它先计算线性得分：

```text
z = w^T x + b
```

再通过 sigmoid 函数转成正类概率：

```text
p(y=1|x) = 1 / (1 + exp(-z))
```

训练时最小化二分类交叉熵：

```text
Loss = - y log(p) - (1-y) log(1-p)
```

在本项目中，逻辑回归的优点是：

- 训练最快。
- 天然输出概率。
- 特征权重可解释性强。

缺点是：

- 决策边界是线性的。
- 对复杂纹理、血管、视盘附近亮斑的区分能力有限。
- 在本项目结果中，Recall 高于 SVM，但 False Positive 也明显更多。

### 7.6 传统模型预测流程

训练完成后，`predict_traditional()` 对测试图像执行：

1. 对整张图计算 14 维特征图。
2. 只取 FOV 内像素。
3. 用 SVM 或 LR 输出每个像素的渗出液概率。
4. 把概率填回二维图像，得到概率图。
5. FOV 外区域概率保持为 0。

二值化阈值不是固定写死，而是在验证集上通过 `best_threshold()` 选择，使 F1 最大。

## 8. U-Net 深度学习算法细节

深度学习部分在 `src/unet_model.py` 中实现。

### 8.1 U-Net 的基本思想

U-Net 是医学图像分割中非常经典的 encoder-decoder 结构。

它包含两条路径：

- 编码器 encoder：逐步下采样，提取高级语义特征。
- 解码器 decoder：逐步上采样，把语义特征恢复到像素级分辨率。

U-Net 的关键是 skip connection。编码器浅层特征会直接连接到解码器对应尺度：

```text
浅层纹理和边界信息 + 深层语义信息 -> 更准确的像素级分割
```

这对医学图像尤其重要，因为病灶通常很小，边界信息很容易在连续下采样中丢失。

### 8.2 本项目的 U-Net 配置

项目使用 `segmentation_models_pytorch` 创建模型：

```python
smp.Unet(
    encoder_name="resnet34",
    encoder_weights="imagenet",
    in_channels=3,
    classes=1,
)
```

含义如下：

- `encoder_name="resnet34"`：使用 ResNet34 作为编码器。
- `encoder_weights="imagenet"`：加载 ImageNet 预训练权重。
- `in_channels=3`：输入是 RGB 三通道。
- `classes=1`：输出一个通道，表示渗出液概率。

由于数据集只有 47 张图像，直接从零训练深度网络很容易过拟合。使用 ImageNet 预训练编码器可以把自然图像中的边缘、纹理、颜色等低层视觉特征迁移过来，降低小数据训练难度。

### 8.3 输入归一化

U-Net 输入图像先归一化到 0-1，然后使用 ImageNet 均值和方差标准化：

```python
imagenet_mean = [0.485, 0.456, 0.406]
imagenet_std  = [0.229, 0.224, 0.225]
```

标准化公式为：

```text
x_norm = (x - mean) / std
```

这样输入分布和 ResNet34 预训练时更接近。

### 8.4 数据增强

训练集在 `SegDS(..., aug=True)` 中进行增强：

- 50% 概率水平翻转。
- 50% 概率垂直翻转。
- 50% 概率旋转 180 度。
- 50% 概率亮度缩放，范围为 `0.85 - 1.15`。

数据增强的作用：

- 扩大有效训练样本。
- 降低模型对固定方向、固定亮度的依赖。
- 缓解 47 张小数据集带来的过拟合风险。

代码中没有使用随机裁剪，实际增强主要是翻转、旋转和亮度扰动。

### 8.5 损失函数：BCE + Tversky

项目定义了组合损失 `ComboLoss`：

```python
loss = 0.5 * BCEWithLogitsLoss(pos_weight=10.0)
     + 0.5 * TverskyLoss(alpha=0.3, beta=0.7)
```

#### 8.5.1 BCEWithLogitsLoss

BCE 是二分类交叉熵。`WithLogits` 表示模型直接输出 logit，损失函数内部会做 sigmoid，因此数值更稳定。

对于一个像素：

```text
BCE = - y log(p) - (1-y) log(1-p)
```

其中：

- `y=1` 表示渗出液。
- `y=0` 表示背景。
- `p` 是模型预测为渗出液的概率。

项目设置：

```python
pos_weight = 10.0
```

这会把前景像素的损失权重放大，缓解前景稀少导致模型倾向预测背景的问题。

#### 8.5.2 Tversky Loss

Tversky 系数是 Dice 的推广形式：

```text
Tversky = (TP + s) / (TP + alpha * FP + beta * FN + s)
Loss = 1 - Tversky
```

项目中：

```python
alpha = 0.3
beta = 0.7
```

因为 `beta > alpha`，FN 的惩罚比 FP 更重。也就是说，模型漏掉真实渗出液会被更严厉惩罚。这适合医学病灶检测，因为漏检通常比多检更危险。

#### 8.5.3 为什么组合 BCE 和 Tversky

BCE 关注每个像素的独立分类是否正确，Tversky 关注整体前景区域的重叠质量。二者结合后：

- BCE 提供稳定的像素级监督。
- Tversky 直接优化前景重叠，适合类别不平衡。
- `pos_weight` 和 `beta=0.7` 共同提升对少数类病灶的关注。

### 8.6 优化器与学习率

训练配置：

```python
optimizer = AdamW(lr=1e-3, weight_decay=1e-4)
scheduler = CosineAnnealingLR(T_max=60)
```

AdamW 的特点：

- 自适应学习率，适合深度网络训练。
- decoupled weight decay 有助于正则化。

余弦退火学习率会让学习率从初始值逐渐下降，有利于训练后期更平滑地收敛。

### 8.7 混合精度训练

代码使用：

```python
torch.cuda.amp.GradScaler()
torch.cuda.amp.autocast()
```

混合精度训练会在部分计算中使用 FP16，从而：

- 降低显存占用。
- 提高 GPU 吞吐。
- 保持梯度缩放来避免数值下溢。

### 8.8 U-Net 模型选择

每个 epoch 训练结束后，代码会在验证集上计算 `val_dice`。

`unet_val_dice()` 会在多个阈值上尝试：

```python
threshold in np.linspace(0.1, 0.9, 17)
```

取验证 Dice 的最大值作为当前 epoch 的表现。如果当前 epoch 的验证 Dice 更高，就保存该模型权重。

Notebook 日志显示：

```text
epoch 45  loss 0.3518  val_dice 0.5325  best 0.5325
epoch 59  loss 0.3312  val_dice 0.5282  best 0.5325
U-Net 学习时间 94.8s  最佳 val_dice 0.5325
```

这说明训练后期 loss 继续下降，但验证 Dice 在 epoch 45 左右达到最佳。最终评估使用的是验证 Dice 最佳的权重，不一定是最后一个 epoch 的权重。

## 9. 算法验证方法

这一部分是答辩重点。建议按“数据不泄漏、阈值不偷看测试集、统一指标、定性和定量同时验证”来讲。

### 9.1 数据划分验证

项目没有把测试集参与训练。

实际流程：

```text
47 张总数据
  -> 32 张题目训练集
       -> 27 张 fit，用于模型参数学习
       -> 5 张 val，用于阈值选择和 U-Net 最佳 epoch 选择
  -> 15 张题目测试集，只用于最终测试指标
```

这可以避免测试集泄漏。尤其是阈值选择也不能在测试集上做，否则会高估性能。

### 9.2 统一预处理和统一评估

SVM、Logistic Regression 和 U-Net 都使用同一批 `test_s`，同一批 `test_gt` 和 `test_fov`。

评价时只在 FOV 内展开像素：

```python
mask = fov > 0
probs = pred_prob[mask]
labels = gt_mask[mask]
```

这样黑色背景区域不会进入指标计算。

### 9.3 阈值选择

模型输出的是概率图，最终分割需要阈值：

```text
prob >= threshold -> 渗出液
prob < threshold  -> 背景
```

项目使用验证集选择阈值，而不是在测试集上调阈值。`best_threshold()` 在以下阈值中搜索：

```python
np.linspace(0.05, 0.95, 19)
```

选择验证集 F1 最高的阈值。

最终得到：

| 方法 | 阈值 |
|---|---:|
| SVM | 0.65 |
| Logistic Regression | 0.95 |
| U-Net | 0.95 |

Logistic Regression 和 U-Net 的最佳阈值都很高，说明在当前类别不平衡任务中，低阈值会带来过多假阳性，需要提高阈值控制误检。

### 9.4 混淆矩阵

项目使用像素级混淆矩阵。矩阵顺序为：

```text
[[TN, FP],
 [FN, TP]]
```

含义：

- TN：背景像素预测为背景。
- FP：背景像素误预测为渗出液。
- FN：渗出液像素漏检为背景。
- TP：渗出液像素预测为渗出液。

由混淆矩阵可以计算：

```text
Accuracy  = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)
Recall    = TP / (TP + FN)
Specificity = TN / (TN + FP)
F1 / Dice = 2TP / (2TP + FP + FN)
IoU       = TP / (TP + FP + FN)
```

对于二值分割，F1 和 Dice 在公式上相同。

### 9.5 ROC 与 PR 曲线

项目同时计算 ROC-AUC 和 PR-AUC。

ROC 曲线：

- 横轴 FPR = FP / (FP + TN)
- 纵轴 TPR = Recall = TP / (TP + FN)
- 反映模型在不同阈值下区分正负类的整体能力。

PR 曲线：

- 横轴 Recall
- 纵轴 Precision
- 更关注正类预测质量。

本项目正类极少，背景极多，因此 PR-AUC 比 ROC-AUC 更有解释力。ROC-AUC 在类别极不平衡时可能看起来不错，因为 TN 数量太大；PR-AUC 会更直接地反映“预测出来的渗出液里有多少是真的”。

`curve_metrics()` 在像素数量过多时最多采样 600000 个像素用于画曲线，但会保留全部正样本，只对负样本下采样。这样既节省计算，又尽量不损失前景信息。

### 9.6 训练过程验证

U-Net 的训练过程由 `fig3_training_curve.png` 验证：

- train loss 从约 0.86 持续下降到约 0.33。
- val Dice 从接近 0 上升到约 0.53。
- 后期曲线趋于平稳，没有出现明显训练崩溃。

这说明模型确实学到了有用的分割特征，而不是随机输出。

### 9.7 定性可视化验证

`fig6_qualitative.png` 展示了测试集中前景比例较高的 4 张样本：

- 原图。
- Ground truth。
- SVM prediction。
- Logistic Regression prediction。
- U-Net prediction。

从图上可以观察到：

- SVM 预测较保守，漏检较多。
- Logistic Regression 更容易产生散点状误检，尤其在血管和视盘附近。
- U-Net 对病灶区域的空间聚集性学习更好，预测区域更接近真实标注。

## 11. 可以直接用于答辩的讲解稿

### 11.1 项目技术路线

本项目首先读取 e-ophtha 眼底图像和专家标注掩膜，并按文件名进行一一配对。由于渗出液只占全部像素的约 0.2%，这是一个极端类别不平衡的像素级分割问题。为此，我先做了 FOV 有效视野提取、光照归一化和 CLAHE 对比度增强，然后分别实现传统机器学习和深度学习两条路线。

传统机器学习路线把每个像素视为一个样本，对每个 FOV 内像素提取 14 维手工特征，包括 RGB、LAB、HSV 颜色特征，局部对比度、局部标准差、Top-hat 和 Sobel 梯度特征，再用 SVM 和 Logistic Regression 进行二分类。深度学习路线使用带 ImageNet 预训练 ResNet34 编码器的 U-Net，直接从图像学习渗出液的空间和纹理特征。

### 11.2 为什么要做这些预处理

眼底图像存在黑色背景、光照不均和局部对比度低等问题。FOV mask 用来排除黑色背景，避免背景像素干扰训练和评价；光照归一化通过中值滤波估计背景亮度，减轻中心和边缘亮度不一致；CLAHE 在亮度通道增强局部对比度，让亮黄色渗出液更突出。

### 11.3 传统 ML 为什么可行

渗出液通常有较强的颜色和亮度特征，比如偏黄、偏亮，并且在局部区域内与周围组织形成对比。因此可以设计颜色、亮度、纹理和形态学特征，然后让线性分类器判断每个像素是否属于渗出液。SVM 更注重最大间隔，预测比较保守；逻辑回归直接建模正类概率，召回更高但误检更多。

### 11.4 U-Net 为什么更好

传统 ML 的特征是人工设计的，难以表达复杂上下文，比如血管、视盘、反光区域和渗出液之间的区别。U-Net 通过编码器提取多层语义特征，通过解码器恢复像素级分辨率，再用 skip connection 保留边缘和局部细节。预训练 ResNet34 编码器还能在小数据集上提供更好的初始化。因此 U-Net 在 Dice、IoU 和 PR-AUC 上显著更好。

### 11.5 我是如何验证算法的

我没有直接在测试集上调参，而是把题目给出的 32 张训练图像进一步划分为 27 张拟合集和 5 张验证集。模型参数只在拟合集上学习，验证集用于选择二值化阈值和 U-Net 最佳 epoch，最后 15 张测试图像只用于最终报告指标。所有模型使用同一套 FOV 内像素评估，统一计算混淆矩阵、Accuracy、Precision、Recall、Dice、IoU、ROC-AUC 和 PR-AUC。同时我还用训练曲线和定性分割图检查模型是否真的学到病灶区域，而不是只依赖背景像素获得高 Accuracy。

### 11.6 最终结论

实验结果显示，U-Net 的 Dice 为 0.5058，IoU 为 0.3385，PR-AUC 为 0.7321，明显优于 SVM 和 Logistic Regression。传统模型训练只需要几秒，但 Dice 只有约 0.09 到 0.10，说明在小病灶分割中手工特征和线性分类器表达能力有限。U-Net 虽然训练约 95 秒，但在误检和漏检之间取得了更好的平衡，更适合眼底渗出液分割任务。

## 12. 答辩中可能被问到的问题

### 问题 1：为什么 Accuracy 很高，但 Dice 不高？

因为渗出液前景像素只占约 0.2%。绝大多数像素是背景，模型只要把背景预测对，Accuracy 就会很高。但分割任务真正关心的是少数前景病灶，所以 Dice、IoU、Recall 和 PR-AUC 更有意义。

### 问题 2：为什么 U-Net 的阈值是 0.95？

阈值是在验证集上搜索得到的，目标是最大化 F1。由于前景非常稀少，低阈值会把很多背景像素误报为渗出液，所以验证集选择了较高阈值来控制 FP。

### 问题 3：SVM 和 Logistic Regression 有什么区别？

SVM 寻找最大间隔超平面，优化 hinge loss，更关注分类边界的间隔；逻辑回归用 sigmoid 输出概率，优化交叉熵。当前结果中 SVM 更保守，误检少但漏检多；Logistic Regression 召回更高，但误检更多。

### 问题 4：为什么要用 PR-AUC？

因为这是极端类别不平衡问题。ROC-AUC 受大量 TN 影响，可能显得乐观；PR-AUC 直接关注正类预测质量，更能反映渗出液检测的实际效果。

### 问题 5：为什么用 Tversky Loss？

Tversky Loss 是 Dice Loss 的推广，可以分别控制 FP 和 FN 的惩罚权重。本项目设置 `beta=0.7`，比 `alpha=0.3` 更大，表示更重视减少漏检，适合医学病灶分割。

### 问题 6：为什么使用预训练 ResNet34？

数据只有 47 张，直接从零训练 U-Net 容易过拟合。ImageNet 预训练的 ResNet34 已经学到通用边缘、纹理和颜色特征，可以提高小数据集训练的稳定性。

### 问题 7：传统 ML 是否完全没价值？

不是。传统 ML 训练速度快、可解释性强，可以作为轻量基线，也能通过特征系数分析了解颜色和亮度特征的重要性。但在复杂医学分割中，它的表达能力不如 U-Net。

## 13. 项目局限与可改进方向

当前项目仍有一些局限：

- 数据量很小，只有 47 张图像。
- 验证集只有 5 张，阈值和模型选择可能有偶然性。
- 当前结果是单次随机划分，没有做多次重复实验或交叉验证。
- 传统 ML 使用线性模型，对复杂非线性边界表达有限。
- U-Net 使用的是整图 resize，可能损失极小病灶细节。
- 没有加入专门的后处理，比如连通域面积过滤、视盘区域抑制、条件随机场等。

可以改进的方向：

- 做 K-fold 交叉验证或多 seed 重复实验。
- 尝试更强编码器，如 EfficientNet、ResNet50、ConvNeXt。
- 使用 Focal Loss、Focal Tversky Loss 或 Dice + BCE 的更多组合。
- 加入 patch-based 训练，提高小病灶分辨率。
- 对传统 ML 加入随机森林、XGBoost、RBF SVM 等非线性模型。
- 加入后处理，过滤孤立小噪点，减少 FP。
- 在更多公开数据集上外部验证泛化能力。

## 14. 最终答辩结论

本项目围绕 e-ophtha 眼底图像渗出液分割任务，构建了从数据读取、预处理、特征提取、模型训练、阈值选择、测试评估到图表导出的完整流程。传统机器学习方法通过 14 维手工特征和线性分类器提供了快速、可解释的基线；深度学习方法通过预训练 ResNet34 U-Net 自动学习多尺度语义和空间结构，在 Dice、IoU、PR-AUC 和 ROC-AUC 上取得明显优势。

最关键的认识是：这个任务不是普通分类准确率问题，而是极端类别不平衡的小目标医学分割问题。因此评价时必须重视 Dice、IoU、Recall、PR-AUC 和混淆矩阵。最终 U-Net 用更高的计算代价换来了更好的病灶分割质量，是当前项目中更适合实际分割需求的方法。
